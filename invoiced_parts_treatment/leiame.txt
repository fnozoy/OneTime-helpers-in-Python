Exemplo de aplicação Backend:

problema: arquivo de faturamento está sendo rejeito por:
    1. número da peça despachada está errado
    2. número da peça pedido está errado
    3. quantidade de peças pedidas está errado

Business
    main_event
        pega o arquivo de pedidos a ser faturado até o final
            pega o número da peça despachada do sitema Wings
            pega o número da peça pedida do sitema Wings
            pega a quantidade de peças do sistema ARPW
            grava o arquivo de saida para o POPIMS

    partnumber
        regras de negócios ref. ao formato do número da peça de acordo com o dicionário de dados de cada aplicação (Wings, POPIMS, OWS)
            converte o formato partnumber do POPIMS 'Basic   /Pref/Su/fix' para o WINGS 'Pref/Basic   /Su/fix'
            converte o formato partnumber do WINGS 'Pref/Basic   /Su/fix' para o POPIMS 'Basic   /Pref/Su/fix'
            converte o formato partnumber do WINGS 'Pref/Basic   /Su/fix' para o OWS PrefBasicSufix

input/output (para informação apenas)
    conecta ao SQL Server DB (WINGS)
    conecta ao Oracle DB (ARPW)
    lê o arquivo txt flat file (POPIMS)... no diretório IO
    grava o arquivo txt flat file (POPIMS)... no diretório IO

DAO (Data Access Object)
    executa query de SELECT no SQL Server (WINGS), para 2 propósitos:
        obter o partnumber despachado
        obter o partnumber do pedido
    executa query de SELECT no ORACLE (ARPW)
        obter a quantidade pedida do partnumber

Bean_TO (transfer object)
    file_register
        Classe Bean para Transfer Object do arquivo txt flat file de input/output
